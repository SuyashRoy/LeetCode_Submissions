# LeetCode Problem 47: Permutations II

## Problem Description
Given a collection of numbers, `nums`, that might contain duplicates, return all possible unique permutations in any order.

### Example:
**Input:** `nums = [1,1,2]`  
**Output:**  
```
[
    [1,1,2],
    [1,2,1],
    [2,1,1]
]
```

### Constraints:
- `1 <= nums.length <= 8`
- `-10 <= nums[i] <= 10`

---

## Approach

The solution uses **backtracking** to generate all unique permutations. Key steps include:
1. **Sorting the Input:** Sorting helps in easily skipping duplicate numbers during the backtracking process.
2. **Tracking Used Elements:** A boolean array is used to track which elements are already included in the current permutation.
3. **Skipping Duplicates:** If a number is the same as the previous one and the previous number has not been used in the current recursive call, it is skipped to avoid duplicate permutations.

The algorithm ensures that only unique permutations are generated by carefully managing the recursion and skipping duplicates.

## Complexity Analysis

- **Time Complexity:** `O(n * n!)`, where `n` is the length of the input array. This accounts for generating all permutations and the cost of copying each permutation.
- **Space Complexity:** `O(n)`, for the recursion stack and the boolean array used to track used elements.
